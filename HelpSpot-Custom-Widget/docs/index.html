Installing and Using the Plugin

To install the plugin simply upload the entire directory (HelpSpot-Custom-Widget/) to your server. Generally, it's recommended that you upload it to the root directory of your server, although this is not necessary.

Initializing the Plugin

You must include (at a minimum) the following code on any page where you want the Widget to display. NOTE: You may need to adjust the file locations depending on where the plugin is located!

			
			<!-- include our stylesheet -->
			<link href="HelpSpot-Custom-Widget/widget.css" rel="stylesheet" type="text/css" media="screen" />
			
			<!-- include jQuery -->
			<script type="text/javascript" src="https://ajax.googleapis.com/ajax/libs/jquery/1.6.2/jquery.min.js></script>
			
			<!-- include widget.js -->
			<script type="text/javascript" src="HelpSpot-Custom-Widget/widget.js"></script>
			
			<!-- setup and initialize the widget plugin -->
			<script type="text/javascript">
			$(function(){
			
			   // you could probably bind this to just about any element...
			   $('html').HelpSpotWidget({
				
			       // the base url to where HelpSpot is installed (***include trailing slash***)
			       host: 'http://support.yourdomain.com/',
				
			       // the base url to where the widget plugin is located (***include trailing slash***)
			       base: 'HelpSpot-Custom-Widget/'
			   });
			});	
			</script>

Customizing the Plugin

A Brief Introduction to Customizing the Plugin

You have the ability to customize many aspects of both the tab and the form. Here is sample of a few of the thgins you can customize:

			<script type="text/javascript">
			$(function(){
			
			   $('html').HelpSpotWidget({
			       host: 'http://support.yourdomain.com/',
			       base: 'HelpSpot-Custom-Widget/',
			       
			       // Customize the Tab
			       //-----------------------------
			       
			           // The type of tab to show (support, feedback, help, questions) 
			           tab_type: 'feedback', 
			           
			           // The color of the tab (accepts hex codes and plain-text)
			           tab_color: '#c00c00', 
			           
			           // Where to place the tab on the page (left, right)
			           tab_alignment: 'right', 
			           
			           // How far from the top of the page to display the tab. (accepts px and %)
			           tab_top: '40%', 
			       
			       // Customize Text Labels
			       //-----------------------------
			       
			           // the text to use on the form submit button
			           submit_button_text: 'Submit', 
			           
			           // The headline that the user sees upon success
			           success_headline_text: 'Your Message Has Been Received!', 
			           
			           // a small blurb of text the user sees after they submit a ticket
			           success_body_text: 'We will review your submission and someone will reply to you soon.' 
			   });
			   
			});	
			</script>

Pre-populating Form Fields with a User's Data

You also have the ability to pre-populate the value of the form fields. This gives you granular control to customize the form per page or even per user

			<?php
			// Presumably, this data would be loaded dynamically from a database or other source 
			// For our demo, we'll just create our own array
			 $user = array(
			    'first_name' => 'John',
			    'last_name' => 'Doe',
			    'email' => 'john@doe.com',
			    'phone' => '555-123-4567'	  
			);
			?>	
			
			<script type="text/javascript">
			$(function(){
			
			   $('html').HelpSpotWidget({
			       host: 'http://support.yourdomain.com/',
			       base: 'HelpSpot-Custom-Widget/',
			       default_first_name: '<?php echo $user['first_name']; ?>',
			       default_last_name: '<?php echo $user['last_name']; ?>',
			       default_email: '<?php echo $user['email']; ?>',
			       default_phone: '<?php echo $user['phone']; ?>'
			   });
			   
			});	
			</script>

Customizing, Creating, and Using Forms

This plugin allows you to create many different forms, and then dynamically set which form gets shown at run time. The main reason for this feature, besides allowing complete customization of the forms, is so that you can create different forms (ie: a form for support, one for suggestions, and one for comments) and then have the appropriate form show depending on what action the user takes.

The first thing you should do is get more familiar with how the forms work. You can do this by examining the default form.php.

Getting Started with Forms -- Things You Should Know!

The most important thing you should know is that the forms must do everything. And by everything, I mean they must print out the actual HTML form, validate the data, submit the data using the HelpSpot API, return any error or success messages, etc. If you have any prior experience with PHP, then this probably won't be nearly as difficult as it sounds.

The next thing you should know is that the forms are actually loaded inside of an iFrame. This presents a few challenges we must overcome:

  • Each form must include its own CSS/Styling -- even if you're using the widget.css stylesheet (which is used by default, and gets loaded at run time), you must still link to the stylesheet again within the form. If you don't want to use an external stylesheet then you can just add your CSS to the page itself using <style> tags.
  • If you plan on using jQuery (or any library) with your form, then you must manually include it (just like the issue with the stylesheets, jQuery must be loaded from within the form, even if it's already been loaded on your site)
  • The iFrame will not auto-resize at this point. I'm still working on it though.
  • I'm pretty sure there was seomthing else, but I can't remember now.

Customizing the Default Form

Customizing the default form involves three files (although you may not have to edit them all):

File Description
form.php This is the form itself. It has all the code to print the HTML form, validate the data, submit the data using the HelpSpot API, etc. This is the file where you would add or remove form fields.
form.js Any custom javascript or jQuery to use with the form
widget.css The default CSS Stylesheet. Currently, this stylesheet has the CSS styles for the entire plugin. In a future version, I'll probably separate the CSS for the form.
In the meantime, assuming that you haven't changed the default CSS file or the div IDs, you can customize the form starting on line 40 of widget.css.

The CSS selectors associated with the form are:

  • #custom_widget_content
  • #cw_form_container   (and all children elements)
  • #frm_cw_form
  • .cw_form   (and all children elements)

Creating and Using a Custom Form

Creating a custom form is much easier if you have prior experience with PHP. If not, you may find the required process a bit intimidating.

It's highly recommended that you start with a copy of the default form.php:

  • Start by creating a new file inside the Helpspot-Custom-Widget directory. For our example, we'll name our file custom_form.php.
  • Next, copy the code from the original form.php and paste it into your newly created file.
  • If you plan on using jQuery, javascript, etc, then you can [optionally] create an external file for this. ie: custom_form.js.

At this point, it's probably best to read over the code in your newly created file (custom_form.php, or whatever you named your file), line-by-line, to get an understanding of what evertything does. Once you've got a good understanding of how it all works, you can start making changes such as adding or removing form fields, and more.

Ready to start making your own forms? Before you start, here's a few things to keep in mind:

  • In order to create new tickets, the forms use cURL to communicate with the HelpSpot API's Public Method request.create. This method has several available parameters, some of which are required for the script to work. Please see HelpSpot's FAQ Page for this method for more information.
  • If you add or remove form fields then be sure to add the code to validate the new fields (or remove the code for any fields you removed).
  • Remember to define your custom form when setting up the plugin (See the code below)

Telling the Plugin to Use Your New Form

			<script type="text/javascript">
			$(function(){
			
			   $('html').HelpSpotWidget({
			       host: 'http://support.yourdomain.com/',
			       base: 'HelpSpot-Custom-Widget/',
			       form: 'custom_form.php'
			   });
			   
			});	
			</script>

Creating Custom Tabs

A .psd file has been included if you would like to create your own custom tabs. Here's how to do it:

  • Locate the .psd template at HelpSpot-Custom-Widget/images/button-template.psd and open it in Photoshop
  • Once in Photoshop, select the text and change it to whatever you want. Keep in mind though, you only have about 95px of height to work with
  • Make sure that no background image is showing (ie: it has a transparent background)
  • Go to File > Save for Web & Devices (or hold down ctrl+alt+shift+s)
  • When the next screen pops up, choose PNG-24 and check Transparency:
  • Next, hit the Save button, which will popup another window where you can name your new image
  • It's recommended that you follow the existing naming convention, which is: button-{button_text}.png
    For example, if the text on your new tab is "FAQs" then you should name your image button-faqs.png (all lowercase)
  • Save the image somewhere to your computer, and then use an FTP program to upload it to HelpSpot-Custom-Widget/images (or wherever you uploaded the plugin files)
  • Edit widget.css and include the following code (this code uses the newly created "FAQs" tab from above. You may need to adjust accordingly)
    						#custom_widget_tab.faqs { background-image: url(images/button-faqs.png); }
  • Finally, you need to tell the plugin to use your new button:
			<script type="text/javascript">
			    $(function(){
			        $('html').HelpSpotWidget({
			            host: 'http://support.yourdomain.com/',
			            base: 'HelpSpot-Custom-Widget/',
	                    tab_type: 'faqs',
			        });
				   
    	    });	
			</script>
